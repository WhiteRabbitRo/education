# Модуль 1 - Базовый синтаксис и система типов

## Теория

* По умолчанию переменные неизменяемы (как `const` в С++).
* Ключевое слово `mut` делает их изменяемыми.
*  **Теневание (shadowing)** - можно объявить новую переменную с тем же именем, она "затеняет" предыдущую.

```rust
let x = 5;          // неизменяемая
let mut y = 10;     // изменяемая
y = 20;

let z = 30;
let z = z + 1;      // теневание
```

### Скалярные типы

* Целые: `i8, u8, i16, u16, i32, u32, i64, u64, i128, u128, isize, usize` (размер зависит от архитектуры).
* По умолчанию `i32` - быстре всего на современных ЦП.
* С плавающей точкой: `f32, f64` (по умолчанию `f64`).
* Логический: `bool` (`true` / `false`).
* Символьный: `char` (4 байта, Unicode).

```rust
let a: i32 = -42;
let b: u32 = 100;
let c = 3.14; // f64
let d = 'A';
```

### Составные типы

* **Кортеж** - фиксированная длина, разные типы. Деструктуризация через `let (x, y) = tuple`.
* **Массив** - фиксированная длина, один тип. Хранится на стеке.
* **Срез (slice)** - динамический вид на массив или вектор. Тип `&[T]`.

```rust
let tuple = (10, 'a', 3.14);
let (first, _, _) = tuple;      // first = 10

let arr = [1, 2, 3, 4];
let slice = &arr[1..3];         // [2, 3]

let x: (i32, f64, u8) = (500, 6.4, 1);
let first = x.0;    // Использование индекса
let second = x.1;
let third = x.2;
let (a, b, c) = x; // Деструктуризация кортежа

let arr_exp = [1, 2, 3, 4];             // Массив из 4 i32 значений
let arr_imp: [u64; 4] = [1, 2, 3, 4];   // Массив из 4 u64 значений
let arr_com = [3; 5];                   // Массив из 5ти значений "3"
```

### Функции

* Объявляются через `fn`.
* Параметры и возвращаемое значение с типом.
* **Выражения** (возвращают значение) - не имеют `;`. **Инструкции** - выполняют действия и не возвращают значения, используют операторы.
* Последнее выражение в функции - неявный `return`.

```rust
fn add(x: i32, y: i32) -> i32 {
    x + y // нет `;` - возвращаем выражение
}

fn main() {
    let result = add(3, 4);
    println!("{}", result);
}

let x = (let y = 6); // Ошибка, инструкция "let y = 6" не возвращает значение
x + y; // Это инструкция-выражение, типом которой становится ()

let w: () = x + y; // Ошибка, поскольку x и y имеют тип i32
let w: () = { x + y; }; // Блок возвращает значение последнего выражения, т.е. ()
```

### Управляющие конструкции

* `if` - выражение, может возвращать значение. Ветки должны возвращать один тип.
* `loop` - бесконечный цил, прерывается `break`.
* `while` - цикл с условием.
* `for` - итерация по диапазону или коллекции.

```rust
let condition = true;
let number = if condition { 5 } else { 10 }; // if как выражение

for i in 0..5 { // 0, 1, 2, 3, 4
    println!("{}", i);
}
```

> Условие `if` должно всегда иметь только логический тип.

### Константы и статические переменные

* `const` - неизменяемая константа времени компиляции (без адреса в памяти, инлайнится).
* `static` - глобальная переменная  с фиксированным адресом (редко используется, требует `unsafe` для изменения).

```rust
const PI: f64 = 3.14159;
let area = PI * r * r; // PI будет подставлена как литерал 3.14159 на этапе компиляции

// если взять адрес let p = &PI; - компилятор может разместить ее в статической памяти
```

> `const` работает как подстановка на этапе компляции (схоже в `#define`), но с безопасной типизацией и без текстовой магии препроцессора. `const` подчинаяется обычным правилам модулей (можно сделать `pub`), т.е. имеет ограниченную область видимости.

#### Особенности теневания

```rust
let x = "42";
x = x.parse::<i32>().unwrap(); // Ошибка, переменная неизменяемая и разные типы

let mut y = "42";
y = y.parse::<i32>().unwrap(); // Ошибка, разные типы

let mut z = "42";
z = "54"; // ОК, одинаковые типы и изменяемая переменная
let mut z = z.parse::<i32>().unwrap(); // ОК, теневание

let a = "42";
let a = a.parse::<i32>().unwrap(); // ОК, теневание
```

#### Метки циклов

```rust
fn main() {
    let mut count = 0;
    'counting_up: loop {                // Создаем метку внешнего цикла
        let mut remaining = 10;
        loop {
            if remaining == 9 {
                break;
            }
            if count == 2 {
                break 'counting_up;     // Выходим из внешнего цикла
            }
            remaining -= 1;
        }
        count += 1;
    }
}
```

#### Срез и массив

* Массив `[T; N]` имеет фиксированный размер, известный на этапе компиляции, т.е. `N` - константа. Срез `[T]` имеет динамический размер, известный только во время выполнения.
* Срез всегда используется через ссылку: `&[T]` или `&mut [T]`.
* Массив может быть на стеке (если не в куче), а срез не владеет данными, только заимствует их.

```rust
let mut arr = [10, 20, 30];
let slice = &mut arr[...];  // Изменяемый срез
slice[0] = 99;              // Меняем исходный массив: arr = [90, 20, 30]
```

* Размер массива в памяти - это размер типа данных умноженный на количество элементов, т.е. `size_of::<[T; N]>()`; Размер среза - это размер указателя + длина.

```rust
let a1 = [1, 2, 3];
let a2 = [0; 10];       // повторение значения 10 раз
let a3: [i32; 0] = [];  // пустой массив

let s1: &[i32] = &[1, 2, 3];
let s2 = &a1[..];       // срез от массива
let s3 = &a1[1..3];     // частичный срез
```

> Часто пишут функции, принимающие срез, чтобы они работали с массивами и векторами одинаково.

## Практика

1. Напишите функцию `is_prime(n: u64) -> bool`, которая проверяет, простое ли число. *Подсказка*: проверяйте делители до `sqrt(n)` (используйте `(n as f64).sqrt() as u64`).
2. Преобразование римских чисел в арабские: напишите функцию `roman_to_int(s: &str) -> i32`, которая принимает строку с римским числом (I, V, X, L, C, D, M) и возвращает целое число. *Пример*: `"XIV" -> 14`. *Правила*: обычно символы складываются, но если меньший стоит перед большим, он вычитается.
3. Игра "Угадай число" (усложненная версия): программа загадывает случайное число от 1 до 100 (используйте крейт `rand` - добавьте `rand = "0.8"` в `Cargo.toml`), пользователь вводит число, программа говорит "больше", "меньше" или "угадал", считает количество попыток, после угадывания спрашивает, хочет ли сыграть еще.
4. Функция `min_max_sum(slice: &[i32]) -> (i32, i32, i32)`: возвращает минимальное, максимально и сумму элементов слайса. Запрещается использовать итераторы (пока тренируем циклы `for`).

## Примечание

> **Ключевые слова** - названия, зарезервированные только для использования в языке.

> По соглашению Rust для именования констант требуется использовать все заглавные буквы с подчеркиванием между словами (например, `const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3;`).

> Rust - **статически типизированный язык** (компилятор должен знать типы всех переменных во время компиляции).

> Каждый вариант целочисленных типов со знаком хранит числа от -(2^(n-1)) до (2^(n-1)), где n - число битов в варианте. Целочисленный тип без знака хранит числа от 0 до 2^n - 1.

> Числовые литералы могут использовать `_` в качестве визуального разделителя (например, 0b1111_0000 или 98_222).

> При возникновении целочисленного переполнения (например, пишем 256 в `u8`) возникает паника в режиме `debug`. В режиме `release` происходит обертывание второго дополнения и паники не возникает, однако тем не менее полагаться на это поведение считается ошибкой. Для обработки возможности переполнения используют другие методы.

> Целочисленное деление обрезает значение в направлении нуля до ближайшего целого.

> **Единичный тип (unit)** - кортеж, не имеющий значений. Записывается как `()` - пустое значение или пустой возвращаемый тип. Схож с `void` в C/C++, но в отличие от него `()` можно использовать как значение.

> **Вектор** - аналогичный массиву тип коллекции, который может увеличиваться или уменьшаться в размере.

> В Rust не важно, где вы определяете свои функции, главное в той области видимости, которую может видеть вызывающий их код (отличие от C/C++).

> Если необходимо раньше выйти из функции и вернуть значение, используем инструкцию `return x;`.

## Ответы на вопросы

1. Выражение `let mut x = 5;` означает определение изменяемой переменной, а `let x = 5` - неизменяемой.
2. Если в `if` в одной ветке вернуть `i32`, а в другой `f64`, то произойдет ошибка при компиляции, поскольку в каждой ветке `if` должен быть представлен одинаковый тип данных.
3. `break` можно использовать для выхода из нескольких вложенных циклов, если для них определены метки, например, если для внешнего цикла определена метка `'first`, то `break 'first;` произведет выход из всех циклов.
4. `const` - константа времени компиляции, ее значение инлайнится в код, а память не выделяется (если не брать адрес), `let mut` - изменяемая переменная на стеке, память выделяется во время выполнения. `const` всегда неизменяема и должна быть инициализирована константным выражением.
