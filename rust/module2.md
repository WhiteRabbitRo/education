# Модуль 2 - Владение, заимствование, времена жизни

## Теория

### Правило владения (ownership)

В Rust у каждого значения есть **владелец** (owner). Правила:

* У каждого значения ровно один владелец.
* Когда владелец выходит из области видимости, значение уничтожается (освобождается память).

```Rust
{
    // s не существует

    let s = String::from("hello"); // s - владелец строки
    // можно использовать s

    // область видимости закончилась, s уничтожается, память освобождается
}
```

### Перемещение (move) vs копирование (copy)

* **Простые типы** (целые, числа с плавающей точкой, `bool`, `char`, кортежи из них) реализуют трейт `Copy`. При присваивании они **копируются**.
* **Сложные типы** (например, `String`, `Vec`) не реализуют `Copy`. При присваивании владение **перемещается**.

```Rust
let x = 5;
let y = x; // x копируется, оба работают

let s1 = String::from("hello");
let s2 = s1; // s1 перемещается в s2, s1 больше не действителен

println!("{}", s1); // ошибка, значение перемещено
```

### Заимствование (borrowing)

Вместо передачи владения можно передать ссылку (`&T` - неизменяемая, `&mut T` - изменяемая).

```Rust
fn calculate_len(s: &String) -> usize {
    s.len()
}

let s = String::from("hello");
let len = calculate_len(&s);
println!("{} {}", s, len);
```

**Правила ссылок:**

* В любой момент времени: либо **одна изменяемая ссылка**, либо **много неизменяемых**.
* Ссылки всегда должны быть валидными (не могут ссылаться на освобожденную память).

```Rust
let mut s = String::from("hello");
let r1 = &s;
let r2 = &s;
let r3 = &mut s; // Нельзя! Уже есть неизменяемые
```

### Срезы (slices)

Срез - это ссылка на непрерывную часть коллекции (без копирования данных).

```Rust
let s = String::from("hello world");
let hello = &s[0..5];
let world = &s[6..11];
```

### Времена жизни (lifetimes)

Времена жизни - это аннотации, которые помогают компилятору проверить, что ссылки не переживают данные, на которые указывают.

Чаще всего компилятор выводит времена жизни автоматически (правила элизии). Но иногда их нужно указывать явно:

```Rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

`'a` - это параметр времени жизни, означающий, что возвращаемая ссылка будет жить не дольше, чем входные.
