# Модуль 0 - Введение и инструментарий

## Теория

### Что такое Rust?

* Системный язык программирования, компилируемый.
* Главная фишка: безопасность работы с памятью без сборщика мусора.
* Достигается через систему владения (ownership), проверяемую на этапе компиляции.
* Производительность на уровне C++, но без утечек памяти и гонок даннах (data races).

### Ключевые принципы

* **Zero-cost abstractions** - высокоуровневые конструкции не замедляют программу.
* **Move semantics** - по умолчанию данные перемещаются, а не копируются.
* **Borrow checker** - гарантирует, что ссылки всегда валидны.

### Инструменты

* ```rustup``` - установщик и менеджер версий.
* ```cargo``` - система сборки, пакетный менеджер, запуск тестов, документации.
* ```rustfmt``` - автоформатирование кода.
* ```clippy``` - линтер (подсказывает идиоматичный код).

### Экосистема

* ```crates.io``` - репозиторий библиотек.
* ```docs.rs``` - документация к любой библиотеке.
* ```rust-analyzer``` - LSP-сервер для редакторов.

## Практика

### 1. Установка Rust

Откройте терминал и выполните:
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

(Windows: скачайте и запустите `rustup-init.exe`)

После установки перезапустите терминал или выполните:
```bash
source ~/.cargo/env     # Linux/macOS
```

Проверьте:
```bash
rustc --version         # например, rustc 1.85.0
cargo --version
```

### 2. Создайте первый проект

```bash
cargo new hello_rust
cd hello_rust
cargo run
```

Вы увидите:
```text
    Compiling hello_rust v0.1.0 (...)
        Finished 'dev' profile [unoptimized + debuginfo] target(s) in 0.98s
            Running 'target/debug/hello_rust'
Hello, world!
```

### 3. Изучите структуру проекта

* `src/main.rs` - код.
* `Cargo.toml` - конфигурация (имя, версия, зависимости).
* `target/` - скомпилированные файлы (пока игнорируем).
* `Cargo.lock` - точные версии зависимостей (автоматически).

Откройте `src/main.rs`:
```rust
fn main() {
    println!("Hello, world!");
}
```

### 4. Измените код

Напишите программу, которая запрашивает имя и здоровается:
```rust
use std::io;

fn main() {
    println!("Как вас зовут?");

    let mut name = String::new();
    io::stdin()
        .read_line(&mut name)
        .expect("Не удалось прочитать строку");

    let name = name.trim(); // удаляем лишний перевод строки
    println!("Привет, {}! Добро пожаловать в Rust.", name);
}
```

Запустите: `cargo run`.
Введите имя - убедитесь, что работает.

### 5. Добавьте инструменты форматирования и линтинга

```bash
rustup component add rustfmt
rustup component add clippy
```

Отформатируйте код:
```bash
cargo fmt
```

Проверьте линтером:
```bash
cargo clippy
```

Если нет предупреждений - отлично.

## Примечание

> **Ассоциированная функция** - это функция, реализованная для типа (например, `String::new()` - здесь `new` является ассоциированной функцией для типа `String`)

> **Изменяемая переменная** - это переменная, созданная с атрибутом `mut` (например, `let mut guess = String::new()`).

> **Изменяемая ссылка** - это ссылка, используемая с атрибутом `mut` (например, `&mut guess`).

> **Заполнитель** - это значение, которое будет напечатано в указанном месте с помощью выражения, заключенного в `{}` (например, ```println!("x = {x} and y + 2 = {}", y + 2)```).

> **Крейт, пакет (crate)** - это коллекция файлов исходного кода Rust.
> **Бинарный пакет** - исполняемый файл.
> **Библиотечный пакет** - код, предназначенный для использования в других программах.

> **Семантическое версирование** - указание номера версии `MAJOR.MINOR.PATCH`, где `MAJOR` - внесение несовместимых изменений в API, `MINOR` - добавление функциональности с обеспечением обратной совместимости, `PATCH` - внесение исправлений с обеспечением обратной совместимости. [https://semver.org/]

> Для добавление пакета используется модуль `[dependencies]` в `Cargo.toml`. После добавления пакета с SemVers нужно ввести команду `cargo build`. При обновлении версии пакета используется `cargo update`.

> Для локальной сборки документации с инструкциями по использованию пакета используется команда `cargo doc --open`.

> **Затенение** - повторное использование имени переменной, чтобы избежать создания двух уникальных переменных.

> **Аннотирование** - явное указание типа переменной через `:` (например, `let guess: u32`).

## Ответы на вопросы

1. Для создания нового проекта с библиотекой необходимо ввести команду `cargo new project --lib` (обычный бинарный проект создается командой `cargo new project --bin`).
2. Если не обрабатывать возвращаемое значение функции, то компилятор показывает предупреждение, но компиляция завершается успешно.
